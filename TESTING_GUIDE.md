# API Testing Guide

## Changes Made

### 1. Fixed Preview Endpoint (`/api/generate/preview`)

**Issue**: The endpoint was generating the PPTX file but returning fake placeholder slide data.

**Fix**: Modified `app/main.py` to:
- Extract the actual `PresentationSpec` generated by the AI
- Return real slide content (title, content, keywords, type) from the spec
- Store both the file path and spec for later use

**Result**: The preview now shows the actual slides that were generated, not placeholders.

### 2. Updated Download Endpoint (`/api/presentation/{presentation_id}/download`)

**Issue**: The presentations_store structure changed to include both path and spec.

**Fix**: Added backward compatibility to handle both:
- New format: `{"path": "...", "spec": {...}}`
- Old format: `"path/to/file.pptx"` (string)

**Result**: Download endpoint now works with the new storage format.

## How to Test

### 1. Check API Keys Configuration

The system needs either OpenAI or Gemini API keys to generate real content:

```bash
# Check your .env file in the project root
cat .env | grep -E "OPENAI_API_KEY|GEMINI_API_KEY"
```

If no API keys are set, the system will use **mock fallback data** which generates generic slides.

### 2. Test Preview Generation

Open your browser console (F12) and watch the Network tab when you click "Generate Preview".

**Request to `/api/generate/preview`:**
```json
{
  "topic": "Your lesson plan text...",
  "slide_count": 10,
  "theme": "Modern Minimal"
}
```

**Expected Response:**
```json
{
  "presentation_id": "uuid-here",
  "topic": "Extracted presentation title",
  "theme": "Modern Minimal",
  "slides": [
    {
      "id": 1,
      "type": "intro",
      "title": "Actual slide title",
      "content": "Actual slide content...",
      "keywords": ["keyword1", "keyword2"],
      "image_url": null
    },
    // ... more slides
  ]
}
```

**What to Check:**
- ✅ Slides array has the correct number of slides
- ✅ Each slide has meaningful title and content (not "Slide 1", "Slide 2")
- ✅ Content matches your input topic
- ✅ presentation_id is returned

### 3. Test Download Function

After generating a preview, click the "Download Presentation" button.

**Request to `/api/presentation/{id}/download`:**
- Method: POST
- Presentation ID from the preview response

**Expected Response:**
- Content-Type: `application/vnd.openxmlformats-officedocument.presentationml.presentation`
- File downloads as `.pptx`
- File opens correctly in PowerPoint/Google Slides/LibreOffice

**What to Check:**
- ✅ File downloads successfully
- ✅ File size is reasonable (should be > 100KB)
- ✅ File opens without errors
- ✅ Slides match the preview content
- ✅ Theme colors are applied correctly
- ✅ Images are included (if Unsplash key is configured)

### 4. Test Different Scenarios

#### A. With Mock Data (No API Keys)
Expected behavior:
- Generic slides based on the topic
- Content like "Core Components", "End-to-End Flow"
- Flow diagram on slide 3
- Summary slide at the end

#### B. With OpenAI/Gemini API Key
Expected behavior:
- Custom slides based on your input
- Relevant titles and content
- Structured lesson plan or presentation
- Keywords extracted from content

#### C. Theme Selection
Test different themes and verify:
- Preview shows correct colors
- Downloaded PPTX has the theme applied
- Text is readable against backgrounds

## Common Issues

### Issue: Seeing generic "Slide 1", "Slide 2" titles
**Cause**: The old code was returning placeholders
**Status**: ✅ FIXED - Now returns actual generated content

### Issue: Preview shows different content than downloaded file
**Cause**: Preview was showing fake data while PPTX had real data
**Status**: ✅ FIXED - Both now use the same PresentationSpec

### Issue: Download returns 404
**Cause**: presentation_id not found in store
**Check**:
- Verify the preview succeeded
- Check server logs for errors during generation
- Ensure the server didn't restart (store is in-memory)

### Issue: PPTX file is corrupted
**Cause**: Error during PPTX generation
**Check**:
- Server logs for Python exceptions
- Verify python-pptx is installed correctly
- Check if output directory is writable

## API Endpoints Summary

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/health` | GET | Check if API is running |
| `/api/themes` | GET | Get available theme presets |
| `/api/templates` | GET | Get available templates |
| `/api/generate/preview` | POST | Generate PPTX and return slide preview |
| `/api/presentation/{id}/download` | POST | Download generated PPTX |

## Debugging Tips

### Enable Verbose Logging
Check your Python server terminal for detailed logs during generation.

### Save Spec JSON
Add to `.env`:
```
SAVE_SPEC_JSON=true
```
This saves a `.spec.json` file alongside each PPTX showing the exact AI-generated structure.

### Check Browser Console
Look for:
- Network errors
- JavaScript errors
- API response data

### Verify File Generation
Check the `output/` directory for generated PPTX files.
